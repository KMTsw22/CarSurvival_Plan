## 🔄 핵심 게임 루프

### 터널 시스템
- 맵마다 **고유 수집 아이템 3개**를 모으면 → **터널 설치 가능**
  - ⚠️ 수집 아이템은 **맵마다 다름** (1챕터 F1 맵 기준 = 🏆 트로피)
- 터널 설치 후 진입 시 → **결계 발동** (이동 가능 공간 한정)
- 결계 안에서 **보스 처치** → 다음 챕터 진입

### ⏱ 10분 타이머 규칙

| 상황 | 결과 |
|------|------|
| 10분 경과 + 터널 **미설치** | ❌ 즉시 게임 오버 |
| 10분 경과 + 터널 **설치 완료** | ✅ 연장 생존 페이즈 돌입 — 터널은 열려 있으며 **플레이어가 원할 때 진입 가능** |

#### 🔥 연장 생존 페이즈 (터널 설치 후)

터널을 설치했다고 해서 바로 들어갈 필요는 없다.  
더 버티면 더 강해지고, 더 강해지면 보스전이 유리해진다. **언제 터널로 뛸지는 플레이어의 판단.**

**몬스터 웨이브 강화 단계 (10분 이후, 1분마다 단계 상승)**

| 단계 | 시점 | 몬스터 강도 | 보상 |
|------|------|------------|------|
| 1단계 | +1분 | 🟢 쉬움 강화 | 경험치 ↑ / 골드 ↑ |
| 2단계 | +2분 | 🟡 보통 강화 | 경험치 ↑↑ / 보물상자 등장 |
| 3단계 | +3분 | 🔴 어려움 강화 | 경험치 ↑↑↑ / 보물상자 ↑ / 골드 ↑↑ |
| 위험 | 이후~ | ☠️ 감당 불가 수준 | — |

> 💡 **플레이어 전략 포인트**  
> 몬스터가 너무 강해져서 감당이 안 된다 싶으면 → **터널로 즉시 런**  
> 조금 더 버틸 수 있다 싶으면 → **더 파밍 후 강화된 상태로 보스 입장**  
> 고위험 고보상 구조 — 욕심 부리다 죽으면 그냥 게임 오버

### 챕터 반복 구조
- 챕터마다 동일한 구조로 반복
- 챕터가 올라갈수록 **수집 아이템 & 연출 변화**로 난이도 시각화

## 🏆 수집 아이템 & 미니보스 시스템

> **맵마다 고유 수집 아이템 3개** — 미니보스 처치로 획득 → 터널 설치 가능  
> 수집 아이템 종류는 맵 테마에 따라 달라짐 (예: 1챕터 F1 맵 → 🏆 트로피)

### 미니보스 등장 타임라인

| 등장 시각 | 처치 보상 | 누적 아이템 |
|----------|----------|------------|
| ⏱ **3분** | 수집 아이템 1개 | 1 / 3 |
| ⏱ **6분** | 수집 아이템 1개 | 2 / 3 |
| ⏱ **9분** | 수집 아이템 1개 | 3 / 3 → 터널 설치 가능 |

### 미니보스 목록

| ID | 이름 |
|----|------|
| MON_003 | 메탈 크러셔 |
| MON_007 | Stoplight Striker |
| MON_005 | Billboard Golem |

> 강화 버전: MON_027 ~ 035

### 전체 흐름

```
게임 시작
    ↓
3분 — 미니보스 1 등장 → 처치 → 수집 아이템 1
    ↓
6분 — 미니보스 2 등장 → 처치 → 수집 아이템 2
    ↓
9분 — 미니보스 3 등장 → 처치 → 수집 아이템 3
    ↓
터널 설치 가능 → 진입 or 연장 파밍 선택
    ↓
10분 초과 — 연장 생존 페이즈 (1분마다 강화: 쉬움 → 보통 → 어려움)
```

### 설계 포인트
- 미니보스를 **못 잡으면 아이템 미획득** → 해당 타이밍 긴장감 자동 생성
- 3분 간격으로 **자연스러운 긴장 사이클** 형성
- 챕터가 올라갈수록 미니보스 HP / 패턴 강화로 난이도 조절
- 미니보스는 일반 몬스터와 **명확히 구분되는 연출** 필요  
  (경고 이펙트, 보스 체력바 표시, BGM 변화 등)

## 📈 성장 시스템

### 스펙업 방법
- **레벨업** — 경험치 획득을 통한 성장
- **공구상자** — 신호등 타워를 처치하면 드랍

### 🚦 신호등 타워 시스템

움직이지 못하는 고정형 원거리 타워. 처치하면 **공구상자 드랍**.  
1챕터 기준 10분 동안 맵에 **총 3개** 등장. 등장 타이밍은 미정.

**신호등 색상별 능력**

| 색상 | 능력 | 효과 |
|------|------|------|
| 🔴 빨간불 | 감속 빔 | 플레이어 이동속도 감소 |
| 🟡 노란불 | 광역 전기 충격 | 범위 내 주기적 전기 데미지 |
| 🟢 초록불 | 데미지 감소 방어막 | 주변 몬스터가 받는 데미지 감소 (자체 생존력 강화) |

> 💡 **설계 포인트**  
> 초록불이 살아있으면 주변 몬스터가 잘 안 죽음 → 초록불 우선 처치 판단 유도  
> 색상마다 위협 유형이 달라 플레이어가 타워 처치 우선순위를 매 판 다르게 결정

### 🎲 희귀 드랍 아이템 (몬스터 처치 시 0.1% 확률)

몬스터를 처치하면 극히 드문 확률로 특수 아이템이 드랍됨. 먹는 즉시 효과 발동.

| 아이템 | 드랍 확률 | 효과 | 지속 시간 |
|--------|---------|------|---------|
| 🧲 자석 | 0.1% | 맵의 모든 경험치를 즉시 플레이어에게 흡수 | 즉시 발동 (일회성) |
| ⛽ 연료 | 0.1% | 주유소 오브젝트 생성 — 주변에 있으면 체력 지속 회복 | 10초간 회복 |

#### 🧲 자석 상세
- 먹는 순간 맵에 흩어진 **모든 경험치 오브젝트**가 플레이어에게 자동 흡수
- 대량의 경험치를 한 번에 회수하는 고효율 아이템
- 레벨업이 연쇄적으로 발생할 수 있음

#### ⛽ 연료 상세
- 먹는 즉시 **주유소 오브젝트**가 현재 위치에 생성
- 주유소 **주변 범위 안**에 있는 동안 체력 지속 회복
- **10초** 동안 회복 가능, 이후 주유소 소멸
- 위험한 상황에서 전략적 위치 선정이 중요

## 🔧 파츠 시스템

### 전체 흐름

```
UPGRADE 버튼 진입
    ↓
┌─────────────────────────┐
│  [파츠 상점]  [내 파츠]  │
└─────────────────────────┘
    ↓              ↓
  가챠 뽑기     보유 파츠 목록
  (골드 소모)   → 파츠 선택
```

## 🎨 PAINT — 외형 커스터마이징 시스템

차량과 캐릭터의 비주얼을 꾸미는 시스템. **스탯 영향 없음 — 순수 외형 변경.**

### 🚗 차량 커스터마이징

| 항목 | 내용 | 해금 조건 |
|------|------|----------|
| 차체 색상 | 단색 / 그라데이션 선택 | 기본 제공 |
| 데칼 / 스티커 | 차체에 부착하는 패턴·로고 | 골드 구매 / 업적 |
| 휠 스킨 | 타이어·휠 디자인 변경 | 골드 구매 |
| 차량 이펙트 | 배기 연기 색상, 부스트 이펙트 | 골드 구매 / 특정 챕터 클리어 |
| 번호판 | 텍스트 커스텀 (닉네임 등) | 기본 제공 |

### 🧑 캐릭터 커스터마이징

| 항목 | 내용 | 해금 조건 |
|------|------|----------|
| 드라이버 스킨 | 헬멧·수트 전체 디자인 변경 | 골드 구매 / 업적 / 챕터 보상 |
| 헬멧 색상 | 헬멧 컬러 변경 | 기본 제공 |
| 수트 색상 | 레이싱 수트 컬러 변경 | 기본 제공 |
| 승리 포즈 | 보스 처치 시 연출 모션 | 특정 업적 달성 |

### PAINT UI 구성

```
[차량 탭] [캐릭터 탭]
    ↓
  미리보기 (탑다운 뷰)
    ↓
  항목 선택 → 색상 팔레트 / 스킨 목록
    ↓
  [확인] [취소]
```

## 🤖 AI 챗봇 시스템 (LFM 기반)

선택한 캐릭터와 실제로 대화할 수 있는 시스템. 캐릭터의 말투·세계관·성격을 **로컬 AI 모델(LFM)** 이 구현.

### 구현 방식 — LFM (Local Foundation Model)

| 항목 | 내용 |
|------|------|
| 모델 방식 | 로컬 실행 (서버 API 호출 없음) |
| 런타임 | llama.cpp / Ollama 번들링 |
| 모델 예시 | Llama 3, Mistral, Phi-3 등 (경량 모델) |
| 인터넷 연결 | 불필요 — 완전 오프라인 작동 |
| API 비용 | 없음 — $5 일회성 판매 구조에 최적 |
| 추가 용량 | 약 2~8GB (모델 파일) |

### 구현 흐름

```
[로비 화면]
    ↓
캐릭터 초상화 클릭
    ↓
채팅 UI 열림 (말풍선 스타일)
    ↓
플레이어 입력
    ↓
캐릭터 페르소나 프롬프트 + 게임 상태 주입
    ↓
LFM 로컬 추론 (오프라인)
    ↓
캐릭터 말투·표정 변화와 함께 응답 출력
```

### 시스템 프롬프트 설계 방식

각 캐릭터마다 고정 시스템 프롬프트 파일로 관리:

```
예시 — 알렉스 프롬프트:
"너는 베테랑 레이서 알렉스야. 말투는 짧고 자신감 있어.
 게임 내 세계관(현재/과거/미래 시대, 몬스터와의 싸움)을 알고 있어.
 플레이어를 파트너로 존중하고, 레이싱 관련 조언을 잘 해줘.
 절대 게임 외적인 현실 이야기는 하지 마."
```

### 챗봇 활용 포인트

| 상황 | 챗봇 반응 예시 |
|------|--------------|
| 게임 전 | "오늘 컨디션 어때? 나는 이미 워밍업 끝났어." |
| 보스 클리어 후 | "그 보스, 쉽진 않았지? 잘 잡았어." |
| 게임 오버 후 | "괜찮아. 다시 가면 돼. 나도 옆에 있을게." |
| 파츠 강화 후 | "엔진 업그레이드? 이제 좀 빨라지겠네." |
| 일반 대화 | 캐릭터 고유 성격으로 자유 대화 |

### 기술 구현 요소

| 항목 | 내용 |
|------|------|
| 모델 | LFM 경량 모델 (로컬 실행) |
| 프롬프트 관리 | 캐릭터별 시스템 프롬프트 파일로 관리 |
| 컨텍스트 유지 | 최근 대화 N턴 단기 메모리 유지 |
| 게임 상태 연동 | 현재 챕터, 보스 처치 여부 등을 프롬프트에 주입 |
| UI | 말풍선 채팅창, 캐릭터 표정 변화 (2~3종 이모션) |
| 사양 요건 | LFM 실행 가능 PC 사양 필요 (저사양 시 챗봇 비활성화 옵션 제공) |
