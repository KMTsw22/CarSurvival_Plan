# Car Survivor — 테이블 설계서 (MVP v2.1)

> 총 10개 테이블 | ID 형식: `{접두어}_{3자리 숫자}` | 참조는 반드시 ID로

---

## 1. TB_Currency (재화)

| 컬럼명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| currency_id | string (PK) | 재화 고유 ID | `CUR_001` |
| currency_name | string | 재화 이름 | 골드 |
| currency_desc | string | 재화 용도 설명 | 업그레이드 비용, 상점 구매 |
| icon_key | string | UI 아이콘 리소스 키 | `ico_gold` |

**고정 데이터:**

| currency_id | 이름 | 용도 |
|-------------|------|------|
| CUR_001 | 골드 | 업그레이드 비용, 상점 구매 |
| CUR_002 | 나사 | 장비 업그레이드 재료 |

## 3. TB_Weapon (무기 — 주무기 + 보조무기)

| 컬럼명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| weapon_id | string (PK) | 무기 고유 ID | `WPN_001` |
| weapon_name | string | 무기 이름 | 기관총 |
| weapon_category | enum | Main=주무기 / Sub=보조무기 | Main |
| base_damage | float | 기본 데미지 (레벨당 증가 기준) | 20 |
| effect_desc | string | 효과 설명 | 마우스 방향 연사 |
| aim_type | enum | Manual=마우스 조준 / Auto=자동 발동 | Manual |
| weapon_type | enum | MachineGun / OilSlick / SawBlade | MachineGun |
| cooldown | float | 쿨타임/발사간격 (초). 0=없음 | 0.3 |
| duration | float | 지속시간 (초). 0=영구 | 0 |
| max_level | int | 최대 레벨 | 5 |
| drop_weight | int | 레벨업 선택지 등장 가중치 | 10 |
| icon_key | string | 아이콘 리소스 키 | `ico_machinegun` |

> **aim_type:** `Manual` = 플레이어가 마우스 커서 방향으로 조준 발사, `Auto` = 자동 발동/자동 타겟팅

**고정 데이터:**

| weapon_id | 이름 | 분류 | 타입 | 조준 | 설명 |
|-----------|------|------|------|------|------|
| WPN_001 | 기관총 | Main | MachineGun | Manual | 마우스 방향 연사 |
| WPN_002 | 오일 슬릭 | Sub | OilSlick | Auto | 기름 장판 설치 (감속) |
| WPN_003 | 회전 톱날 | Sub | SawBlade | Auto | 차량 주변 회전 데미지 |

## 8. TB_Wave (웨이브 타임라인)

| 컬럼명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| wave_id | string (PK) | 웨이브 고유 ID | `WAVE_001` |
| wave_group_id | string | 웨이브 그룹 ID. TB_Map에서 참조 | `WG_CH1` |
| time_min | int | 시작 시간 (분) | 0 |
| phase_name | string | 페이즈 이름 | 튜토리얼 |
| mon_id | string (FK) | → TB_Monster.mon_id | `MON_001` |
| spawn_per_30s | int | 30초당 스폰 수 | 4 |
| max_enemies | int | 동시 최대 적 수 | 15 |
| difficulty_scale | float | 난이도 배율 (몬스터 스탯에 곱) | 1.0 |
| note | string | 비고 | 시작, 타이어 좀비만 |

**참조 관계:** `mon_id` → `TB_Monster`

> `wave_group_id`로 웨이브를 그룹화. 여러 맵이 같은 그룹을 공유할 수 있음 (예: 다른 맵에서 같은 몬스터 웨이브 사용)
> 같은 시간대에 여러 적이 스폰되면 **행이 여러 개** 생김 (예: 2분대에 타이어 좀비 + 오일 슬라임 = 2행)

**웨이브 구조 (WG_CH1 — 10분 기본 + 연장 생존 페이즈):**

| 시간 | 페이즈 | 등장 몬스터 |
|------|--------|-------------|
| 0분 | 튜토리얼 | MON_001 |
| 1분 | 초반 | MON_001 |
| 2분 | 초반 | MON_001 + MON_002 합류 |
| 3분 | 중반 진입 | MON_001~003 (메탈 크러셔 합류) |
| 4~5분 | 중반 | 전 몬스터 + MON_004 합류 |
| 6분 | 중반 후반 | 밀도 증가 |
| 7~8분 | 후반 | 고밀도 |
| 9분 | 트로피 러시 | 트로피 수집 마감 |
| **10분** | **연장 1단계** | 쉬움 강화 — 경험치/골드 UP |
| **12분** | **연장 2단계** | 보통 강화 — 보물상자 등장 |
| **14분** | **연장 3단계** | 어려움 강화 — 고보상 |
| **16분~** | **위험** | 감당 불가 수준 |

> 10분 경과 시 터널 미설치 → 게임 오버 / 터널 설치 완료 → 연장 생존 페이즈 진입

## 10. TB_Map (맵)

| 컬럼명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| map_id | string (PK) | 맵 고유 ID | `MAP_001` |

| map_desc | string | 맵 설명 | 현재 시대 - F1 경기장 |
| chapter | int | 챕터 번호 | 1 |

## 테이블 관계도 요약

```
TB_Currency ←──── TB_Car.unlock_currency_id

TB_Weapon (레벨업 시 무기 선택지)
TB_SpellBook (레벨업 시 마법서 선택지)
TB_Part (빈 테이블 — 향후 사용)

TB_Monster ←──── TB_MonsterDrop.mon_id
    ↑
    └──── TB_Wave.mon_id
              ↑
TB_Map ──────┘  (wave_group_id로 웨이브 그룹 참조)

TB_Level (레벨별 경험치 / 난이도 배율)
```

## 데이터 무결성 체크리스트

- [ ] TB_Map의 wave_group_id가 TB_Wave에 1개 이상 존재하는가
- [ ] TB_Wave의 mon_id가 TB_Monster에 존재하는가
- [ ] TB_MonsterDrop의 mon_id가 TB_Monster에 존재하는가
- [ ] TB_Part의 drop_weight 합계가 의미있는 값인가
- [ ] TB_Car.unlock_currency_id가 TB_Currency에 존재하는가
- [ ] TB_MonsterDrop에 모든 몬스터의 드롭 정보가 있는가
