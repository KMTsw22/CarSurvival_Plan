# TB_Wave (웨이브 타임라인 테이블)

> 런 중 시간대별로 어떤 적이, 얼마나, 어떤 난이도로 스폰되는지를 정의합니다.

## 컬럼 정의

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `wave_id` | string **(PK)** | 웨이브 고유 ID. 형식: `WAVE_###`. 같은 시간대라도 적 종류별로 행이 나뉨 |
| `time_min` | int | 웨이브 시작 시간 (분). 0 = 게임 시작 |
| `phase_name` | string | 구간 이름 (UI/디버그용). 튜토리얼/초반/중반/후반/위기/보스전/런종료 |
| `mon_id` | string **(FK)** | 이 웨이브에서 스폰하�� 적 ID. → `TB_Monster.mon_id` |
| `spawn_per_30s` | int | 30초당 스폰 수. 0 = 추가 스폰 없음 (보스전 등) |
| `max_enemies` | int | 해당 시간대 동시 존재 가능한 최대 적 수 |
| `difficulty_scale` | float | ��이도 배율. 적 HP/속도에 곱해짐. 1.0 = 기본 |
| `note` | string | 특이사항 메모 |

## 웨이브 구간 요약

| 시간 | 구간 | 등장 적 | 난이도 배율 |
|------|------|---------|-----------|
| 0~1분 | 튜토리얼/초반 | 러스티 세단 | 1.0~1.1 |
| 2~6분 | 초반~중반 | + 스파이크 바이크, 장갑 픽업 | 1.2~1.6 |
| 7~11분 | 후반~위기 | + 불꽃 버기 | 1.8~2.3 |
| 12~14분 | 보스전 | 워로드 탱크트럭 단독 | 3.0 |
| 15분 | 런 종료 | 없음 | 0 |

## 데이터 구조 참고

- **같은 시간대에 여러 적**이 스폰되면 행이 여러 개 생김
  - 예: 2분대 → `WAVE_002` (러스티 세단 5마리) + `WAVE_003` (스파이크 바이크 2마리)
- `max_enemies`와 `difficulty_scale`은 같은 시간대의 모든 행에서 동일한 값

## 비고

- 런 1회 목표 시간: 10~15분
- 보스 등장(12분) 시 기존 적 전부 제거
- 15분(연료 소진) 시 강제 런 종료
